00 Architecture Overview
How It Works

Mỗi ảnh món ăn trong app được tạo qua 4 layer: (1) System Prompt — lock toàn bộ style + hướng dẫn AI cách dùng reference images, không bao giờ thay đổi. (2) Reference Images — 5 ảnh thật từ web, tự động thu thập per-dish, cho AI thấy nguyên liệu thật trông thế nào. (3) User Prompt — chỉ đạo composition và khóa garnish, 1-3 câu ngắn. (4) CSS Post-Process — light blur + warmth. Tỷ lệ đóng góp: system prompt 50% (style + reference instruction), reference images 30% (ingredient truth), user prompt 15% (composition + garnish), CSS 5%.

Layer 1 · System Prompt

Locked. Controls framing, lighting, lens, food styling philosophy, strict rules, and reference image instruction. Set once in pipeline. Never modified per-dish.

Layer 2 · Reference Images

Per-dish. 5 ảnh thật từ web. Mục đích duy nhất: cho AI thấy nguyên liệu thật trông thế nào — đúng loại, đúng màu, đúng texture khi nấu chín. Không dạy composition hay photography.

Layer 3 · User Prompt

Per-dish. 1-3 câu ngắn. Chỉ đạo composition (hero là gì, ẩn gì) và khóa garnish. Không mô tả ingredient — references đã xử lý.

Layer 4 · CSS Post-Process

Global. Applied via CSS class on every food image in the app. Soft blur + warm shift. Removes AI crispness, aligns with UI palette.

Image Dimensions — Pipeline Defaults
Context Aspect Ratio Generation Size Notes
Swipe card 3:4 portrait 1024 × 1365 Primary use. Portrait fits card layout, object-fit:cover crops sides
Recipe detail hero 1:1 square 1024 × 1024 50/50 split layout. Square gives max flexibility for cover crop
Thumbnail / list 1:1 square Resized from hero Don't re-generate. Downscale hero image to 256×256
Social share / OG 16:9 landscape 1024 × 576 Optional. Generate separately if needed for marketing
Pipeline Rule

Generate 2 images per dish: one 3:4 (swipe card) and one 1:1 (hero/thumbnail). Do not generate landscape unless explicitly needed for marketing. Thumbnail is always a downscaled crop from the 1:1, never a separate generation.

01 Reference Images — Ingredient Authenticity
Purpose — Narrow

References exist for one reason only: để nguyên liệu trong ảnh AI trông giống thật. AI không biết thịt cá lóc kho trông thế nào, nước canh chua màu gì, bún bò Huế có gân bò ra sao. 5 ảnh thật per dish cho AI thấy: nguyên liệu đúng loại, đúng màu, đúng texture khi đã nấu xong. Mọi thứ khác — framing, lighting, vessel, composition — vẫn do system prompt kiểm soát 100%.

Scope Lock

References KHÔNG teach composition, portion size, photography angle, vessel choice, lighting, hoặc background. Những thứ đó đã locked trong system prompt. Nếu AI bắt đầu copy style chụp ảnh từ references → system prompt block REFERENCE IMAGES chưa đủ mạnh, cần reinforce "ignore everything except ingredients."

02 System Prompt — Locked
Không Sửa

System prompt dưới đây là final. Không thay đổi theo từng món. Mọi variation nằm ở user prompt. Nếu cần sửa system prompt, phải test lại toàn bộ 6 dish categories (A–F) trước khi deploy.

SYSTEM PROMPT

You are a food photography director. Generate images following
this exact style specification for every dish:

FRAMING:
Extreme close-up. A warm brown or cream stoneware bowl/plate,
cropped so tight the rim is cut off at the bottom and sides
of the frame — only the interior of the dish is visible. Shot
from 25 degrees, almost level with the rim. Crumpled natural
linen barely visible at top edge. Background must contain
physical texture — visible linen weave, wood grain, or the
edge of another object slightly out of focus. Never a smooth,
clean, gradient-like background. The background should look
like a real kitchen table, not a studio backdrop.

FOOD STYLING:
Generous, overflowing portions — never minimalist plating.
Everything looks wet, moist, glistening, freshly served, still
settling. Visible oil sheen on broth/sauce surfaces catching
the light. Ingredients tangled naturally, messy, overlapping —
real homemade food, not restaurant-plated. The food must look
delicious and appetizing above all else.

LENS & FOCUS:
The image looks slightly out of focus everywhere — as if the
autofocus missed slightly. Fine details like pepper granules,
herb veins, and meat texture should blur together, not be
individually resolved. Think f/1.8 focus missed by 2cm. The
image should feel like it was captured quickly, not carefully
focused. Never crisp, never tack-sharp.

LIGHTING:
Warm backlight from behind and slightly right. Golden-amber
tones throughout. Muted background, but the food itself stays
vibrant and colorful.

REFERENCE IMAGES:
If reference images are provided, they show what the real
ingredients of this dish look like when cooked.
Study them ONLY for:
— What the correct ingredients are
— What color the ingredients become when cooked
— What texture the surfaces have (glossy, matte, fibrous,
  translucent, etc.)
— What color and opacity the broth or sauce is
— How the ingredients are cut or shaped

IGNORE everything else from the references: the photography
angle, lighting, background, bowl/plate type, plating
arrangement, and composition. These are controlled by the
rules above. Generate ingredients that look like the
references, but photographed entirely in our style.

STRICT RULES:
No people. No hands. No text. No watermark. No cilantro or
scallions unless specified. Maximum 2-3 garnish elements per
dish — keep it simple. Never use green-tinted or white
ceramic. Never produce a smooth gradient background. The ONLY
garnish visible is what the user prompt specifies — nothing
else whatsoever.
03 User Prompt — Composition & Garnish Only
Role Shift — V2

Với reference images, AI đã thấy nguyên liệu trông thế nào. User prompt không cần mô tả ingredient nữa — không cần "glossy dark caramel sauce," "pink rare beef," hay "golden-orange tamarind broth." Những thứ đó AI đã thấy trong ảnh thật. User prompt giờ chỉ còn 2 việc: (1) chỉ đạo composition — cái gì là hero, cái gì ẩn, cái gì nổi — và (2) khóa garnish — liệt kê chính xác garnish cho phép, viết "only" để ngăn AI thêm.

Template

Mỗi user prompt = 1-3 câu ngắn. Cấu trúc: [Tên món] → [Composition directive: hero là gì, ẩn gì, tỷ lệ gì] → [Garnish lock: liệt kê + "only"]

Category Composition Rules — 6 Loại Món
A · Món Nước / Canh / Soup

Phở, Bún Bò, Canh Chua, Hủ Tiếu, Mì Quảng...

Composition: "Most ingredients hidden beneath the broth." Chỉ 1 protein nổi trên mặt. Nước dùng là hero — chiếm phần lớn visible area.

Garnish lock: 1-2 items + "only"

B · Món Kho / Braised

Cá Kho Tộ, Thịt Kho Tàu, Sườn Kho, Gà Kho Gừng...

Composition: Sauce pooling at bottom, protein glistening on top. Vessel override nếu cần: "in a small clay pot."

Garnish lock: 1-2 lát ớt + tiêu + "only"

C · Món Xào / Stir-Fry

Rau Muống Xào Tỏi, Bò Xào, Gà Xào Sả Ớt, Mực Xào...

Composition: "Piled high as if just tossed from a wok." Không flat, không arranged — messy, overlapping, abundant.

Garnish lock: Món xào tự có garnish trong chính nó. Chỉ thêm 1 lát ớt nếu cần.

D · Món Hấp / Luộc / Nhẹ

Cá Hấp, Gà Luộc, Rau Luộc, Canh Rau...

Composition: Protein centered. "Small dipping bowl of fish sauce beside it" nếu có nước chấm.

Garnish lock: Hành lá + gừng thái sợi + "only"

⚠ Exception: Category D is the only category where scallion/hành lá is permitted. Write it explicitly in the user prompt to override the system-level "no scallions" rule.

E · Cơm / Bát Cơm

Cơm Tấm, Cơm Gà, Cơm Sườn, Cơm Chiên...

Composition: "Protein is the hero, rice is the base." Protein chiếm focus, cơm nền phía dưới.

Garnish lock: Đồ chua + 1 lát ớt + "only"

F · Món Cuốn / Gỏi / Cold

Gỏi Cuốn, Bò Cuốn Lá Lốt, Gỏi Đu Đủ, Nem Rán...

Composition: "Cut in half showing the layers inside." Nước chấm bên cạnh.

Garnish lock: "Lettuce leaves beside" + "only"

User Prompt Checklist

Mỗi user prompt phải pass qua checklist này trước khi submit:

Tên món tiếng Việt — dòng đầu tiên, in đậm
Composition directive — hero là gì, cái gì ẩn, "piled high" / "hidden beneath broth" / "cut in half"
Garnish lock — liệt kê chính xác items cho phép, kết thúc bằng "only"
Vessel override (nếu cần) — chỉ khi khác default (vd: "in a small clay pot" cho cá kho)
Không Làm

Không mô tả ingredient appearance trong user prompt. Không viết "glossy dark caramel sauce," "rich golden broth," "pink rare beef," "charred grill marks." References đã xử lý — viết thêm là thừa và có thể conflict với ảnh thật.

05 Pitfalls & Fixes — Các Lỗi Thường Gặp
✕ Sai
Mô tả ingredient trong user prompt khi đã có references → redundant, có thể conflict với ảnh thật
Liệt kê nhiều nguyên liệu trên mặt soup → model hiển thị tất cả, trông cluttered
Viết "no cilantro, no scallions" → negative prompt yếu, model vẫn thêm
User prompt quá dài (4+ câu) → model confused giữa references và text
Dùng "minimalist" hoặc "clean plating" → phần ăn trông ít, không hấp dẫn
Reference ảnh kém chất lượng (mờ, filter nặng, sai món) → AI học sai ingredient
✓ Đúng
User prompt chỉ nói composition + garnish lock. References lo ingredient truth
Viết "most ingredients hidden beneath the broth" — chỉ show 1-2 thứ nổi
Viết "one basil sprig and two chili slices only" — positive + "only" enforces
Giữ user prompt 1-3 câu. Composition directive + garnish lock. Hết.
Viết "generous, overflowing portion" — system prompt đã enforce nhưng reinforce thêm
Chọn reference ảnh rõ nguyên liệu, màu tự nhiên, từ food blogs tiếng Việt
Known Model Limitations

Có 2 thứ model AI luôn kháng cự dù prompt nào: (1) Sharpness — model mặc định tạo ảnh hyper-crisp. Prompt soft focus chỉ giảm ~50%. Giải quyết bằng CSS post-process layer. (2) Extra garnish — model thêm cilantro/scallion dù không yêu cầu. Dùng positive-only language ("only") thay vì negative. Chấp nhận ~80% compliance rate, re-generate nếu cần.

06 CSS Post-Process Layer
Rationale

CSS filter áp dụng trên mọi ảnh món ăn trong app. Mục đích: (1) Giảm AI crispness bằng subtle Gaussian blur (1.5–2px). (2) Tăng nhẹ warmth và contrast. (3) Đảm bảo ảnh blend tự nhiên với UI palette. Filter cực nhẹ — không nên nhìn thấy bằng mắt thường. Nếu filter visible, nó quá mạnh.

CSS Implementation
/* ── FOOD PHOTO POST-PROCESS ──
   Apply .food-photo to every AI-generated food image.
   This is a GLOBAL class — not per-dish. ── */

.food-photo {
  /* Subtle blur to reduce AI crispness + warm shift + micro contrast
     blur(1px) = softens AI hyper-sharpness
     brightness(1.02) = 2% brighter, avoids dull look from blur
     contrast(1.04) = 4% more contrast, restores depth
     saturate(1.03) = 3% more saturated, compensates blur wash-out */
  filter: blur(1px) brightness(1.02) contrast(1.04) saturate(1.03);

  /* Prevent blur from bleeding outside image bounds */
  overflow: hidden;
  border-radius: inherit;

  /* GPU acceleration — ONLY on swipe cards (animated).
     Do NOT apply will-change globally — it allocates a GPU
     compositor layer per element. On history screens with
     30+ thumbnails, this causes memory issues on low-end
     Android. Only use on elements that animate. */
  transform: translateZ(0);
}

/* ── SCALE COMPENSATION ──
   blur(1px) slightly shrinks perceived image size.
   scale(1.01) compensates — prevents thin transparent edge. ── */

.food-photo img,
.food-photo-img {
  display: block;
  width: 100%;
  height: 100%;
  object-fit: cover;
  transform: scale(1.01);
}

/* ── VARIANT: SWIPE CARD ──
   Rounded corners + animation perf.
   will-change ONLY here — swipe cards are the only
   animated food images. Do not add to thumbnails. ── */

.swipe-card .food-photo {
  border-radius: var(--r-xl);  /* 24px — card corner radius */
  overflow: hidden;
  will-change: filter, transform;
}

/* ── VARIANT: FULL-BLEED HERO (Recipe Detail) ──
   Full-height photo panel, no rounded corners.
   Slightly stronger blur OK since image is larger. ── */

.recipe-hero .food-photo {
  filter: blur(1.5px) brightness(1.02) contrast(1.04) saturate(1.03);
  border-radius: 0;
}

/* ── VARIANT: THUMBNAIL / HISTORY LIST ──
   Small images (< 120px). Blur must be lighter. ── */

.food-thumbnail .food-photo {
  filter: blur(0.5px) brightness(1.02) contrast(1.03) saturate(1.02);
}

/* ── DO NOT apply .food-photo to:
   - UI screenshots
   - Icons / illustrations
   - User-uploaded photos (already real)
   - Non-food imagery
   Only AI-generated food images. ── */
Filter Values — Why These Numbers
Property Value Rationale
blur 1px (card) · 1.5px (hero) · 0.5px (thumb) Removes AI hyper-sharpness. 1px = barely visible trên phone. Hơn 2px = trông lỗi, không phải style
brightness 1.02 (+2%) Compensate blur darkening. Blur averaging lowers perceived brightness — 2% recovers it
contrast 1.04 (+4%) Blur flattens local contrast. 4% boost restores depth without clipping highlights
saturate 1.03 (+3%) Blur desaturates slightly (averaging). 3% compensates. Giữ food colors vibrant
scale 1.01 (on img) Blur creates ~1px transparent edge. Scale eliminates it. Invisible at 1%
Không Tăng

Nếu ảnh trông "chưa đủ mềm," không tăng blur. Tăng blur trên 2px sẽ trông lỗi trên mobile. Thay vào đó, re-generate ảnh với prompt mạnh hơn. CSS layer chỉ shave off 10-15% AI crispness — không thay thế prompt quality.

07 Photo–UI Integration — Blend-In Rules
Core Principle

Ảnh và UI share cùng color DNA. Không cần gradient overlay, không cần CSS blend mode. Ảnh tự blend vì prompt đã tạo ảnh có tông màu trùng với design tokens: linen = var(--bg) #FEF8F1, wood table = var(--text-1) #1E1410, food warmth = var(--primary) #D9622B. Hard edge giữa ảnh và UI là đúng — giống Food Mood.

Photo Edge → UI Token Mapping
Photo Edge Zone Natural Color UI Token Match
Linen cloth (top/top-right) Warm cream, off-white var(--bg) #FEF8F1 — exact match
Wood table (bottom/sides) Dark walnut brown var(--text-1) #1E1410 — near match
Bowl rim Warm brown/cream ceramic var(--border-strong) #D4B8A5 — harmonic
Food warmth (overall) Golden-amber cast var(--primary) #D9622B — same warmth family
Blurred background Soft warm brown/cream bokeh var(--surface-alt) #F7EDE0 — harmonic
Layout Rules — Where Photo Meets UI
Recipe Detail

50/50 split. Photo sticky left, content scroll right. Hard edge, no gradient. Photo fills 100vh height, object-fit: cover. Ảnh bị crop ở right edge — OK, miễn center dish visible.

Swipe Card

Photo top, info bottom. Photo chiếm ~60% card height. border-radius: var(--r-xl). Hard edge at bottom where photo meets card body. Gradient overlay ở bottom 30% of photo for dish name text readability — CHỈ dùng gradient ở đây.

History / List

Thumbnail left, text right. 80×80px hoặc 64×64px. border-radius: var(--r-md). Ảnh nhỏ nên dùng .food-thumbnail variant (blur 0.5px). object-fit: cover, focus center.

✕ Không làm
Gradient overlay từ ảnh sang UI background
CSS blend-mode trên ảnh (multiply, overlay, etc.)
